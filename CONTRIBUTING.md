# Contributing

When opening a PR:

* explain what changed
* explain why the change should exist
* avoid mixing unrelated changes together
* include screenshots for UI changes
* include a short video/GIF for interaction or animation changes when relevant

If you are planning a larger feature or architectural change, opening an issue first is recommended.

We may close PRs, ask for changes, request a smaller scope, or reimplement ideas ourselves later.
